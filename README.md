# ♻️ EcoSort — Smart Waste Classification

> Snap a photo of an item and EcoSort tells you which bin it belongs in — and shows you *why*.

EcoSort is a computer-vision system that classifies images of household waste into
six recyclability streams (**cardboard, glass, metal, paper, plastic, trash**) and
returns an actionable disposal recommendation with a Grad-CAM explanation. It was
built from scratch for the AIPI 540 Computer Vision module project.

**Live demo:** _see the deployed link in the repository's GitHub "About" section_
**Dataset:** [TrashNet](https://github.com/garythung/trashnet) (~2,500 labeled images)

---

## Why this project?

Recycling contamination — putting the wrong item in a recycling bin — causes entire
loads to be sent to landfill. The decision of *which bin* is genuinely hard for
consumers. EcoSort turns that decision into a one-tap action and, unlike a plain
classifier, it (1) **explains** its prediction with a heat-map and (2) is benchmarked
for **robustness** to the messy, real-world photos people actually take.

### What's novel here (vs. prior TrashNet work)

Most published TrashNet projects report a single CNN accuracy number. EcoSort adds:

1. **A corruption-robustness benchmark** comparing a classical model and a deep model
   under five realistic image corruptions (noise, blur, brightness, JPEG, occlusion) at
   increasing severities — the kind of degradation real phone photos exhibit.
2. **Built-in explainability** (Grad-CAM) surfaced directly in the product UI.
3. **A product-grade, deployable web app** (not just a notebook) running CPU-only inference.

## Three modeling approaches (all included)

| # | Approach | What it is | Where to find it |
|---|----------|------------|------------------|
| 1 | **Naive baseline** | Majority-class predictor (`DummyClassifier`) | [`scripts/naive_model.py`](scripts/naive_model.py) |
| 2 | **Classical ML** | Random Forest over HSV colour-histogram + HOG features | [`scripts/classical_model.py`](scripts/classical_model.py), features in [`scripts/build_features.py`](scripts/build_features.py) |
| 3 | **Deep learning** *(deployed)* | Fine-tuned MobileNetV3-Small (ImageNet transfer learning) | [`scripts/deep_model.py`](scripts/deep_model.py) |

The **deep model is the deployed model**. All three are trained, evaluated on the same
held-out test set, and compared in the report.

## Setup & run

Requires Python 3.11+.

```bash
# 1. Install dependencies
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# 2. Reproduce everything (download data, train all models, evaluate, run experiment)
python setup.py          # add --quick for a 2-epoch smoke test

# 3. Run the web app locally
python main.py           # open http://localhost:8000
```

Or use the Makefile: `make all`, `make app`, `make evaluate`, `make experiment`.

### Individual steps

```bash
python scripts/make_dataset.py     # data
python scripts/build_features.py   # classical features
python scripts/model.py            # train all 3 models
python scripts/evaluate.py         # metrics + plots
python scripts/experiment.py       # robustness experiment
```

---

## Deployment

The app is containerized (CPU-only PyTorch) and deployed to **Railway** via the
included `Dockerfile`. It serves **inference only** — no training happens in the app.
Health check: `GET /api/health`. Prediction API: `POST /api/predict` (multipart field
`image`).
